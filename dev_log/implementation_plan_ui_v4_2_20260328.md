# Implementation Plan: Post-Processing UI v4.2 Upgrade

사용자 피드백을 반영하여 Post-Processing UI의 명칭을 정립하고, 그래프의 물리적 가독성과 테마 제어 기능을 강화합니다.

## 1. 주요 변경 사항 (Key Changes)

### 1.1. 명칭 및 구조 변경
- **탭 명칭**: '기구학 분석' -> **'모션 분석 (Motion Analysis)'**으로 변경.
- **백앤드 설정**: 상단 풀다운 메뉴의 백엔드 변경 옵션을 삭제하고, 기본 백엔드를 **PySide (QtAgg)** 기반으로 설정.
- **메뉴 보강**: Matplotlib 테마(style)를 동적으로 변경할 수 있는 메뉴 추가.

### 1.2. 모션 분석(구 기구학) 탭 강화
- **좌표계 선택**: 'Global(절대)'과 'Local(상대)' 옵션 추가 (기본값: Global).
- **일괄 선택**: '1. 데이터 종류' 및 '3. 추출 위치' 섹션에 **전체 선택 / 전체 해제** 버튼 추가.
- **데이터 로직**: 
    - Global 선택 시: 월드 좌표($P(t)$) 그대로 출력.
    - Local 선택 시: 시작점 기준 변위($P(t) - P(0)$) 출력.

### 1.3. 그래프 품질 개선 (Chart Quality)
- **축 레이블(Label)**: 모든 그래프에 X축(`Time (s)`) 및 Y축(단위 포함 물리량) 레이블 강제 적용.
- **범례(Legend)**: Matplotlib 호환성을 고려하여 `framealpha` 사용 및 스타일 최적화.

## 2. 세부 구현 작업 (Task List)

### 2.1. [UI] 사이드 메뉴 및 탭 명칭 업데이트
- `_build_side_menu_buttons` 및 관련 텍스트 수정.

### 2.2. [UI] 모션 분석 탭 레이아웃 재구성
- '1. 데이터 종류' 하단에 전체 선택 버튼 추가.
- '3. 추출 위치' 하단에 전체 선택 버튼 추가.
- '좌표계 선택' 라디오 버튼 프레임 추가.

### 2.3. [UI] 테마 선택 기능 추가
- `plt.style.available` 목록을 활용한 상단 메뉴 구축.

### 2.4. [Logic] 좌표계 처리 로직 고도화
- `_coord_system_var` (tk.StringVar) 추가 및 `_get_kinematic_series` 연동.

### 2.5. [Logic] Matplotlib 백엔드 고정
- `pyside6` 또는 `PySide2` 등 Qt 환경에 최적화된 백엔드 호출 코드 적용.

## 3. 검증 계획
1. **실행 확인**: UI가 정상적으로 뜨고 PySide 백엔드 윈도우가 열리는지 확인.
2. **Global 기능 확인**: 낙하 시작 시 Z 좌표가 0이 아닌 낙하 높이(0.5m 등)에서 시작하는지 확인.
3. **일괄 버튼 확인**: 8개 코너와 데이터 종류 체크박스가 한 번에 제어되는지 확인.
4. **테마 변경 확인**: 메뉴 선택 시 그래프 창의 색상 테마가 즉시 반영되는지 확인.
