# Walkthrough - Dashboard & Analysis Engine Stabilization (Final)

## 1. 개요
`plate_by_markers_v2.py`의 리팩토링 과정에서 발생한 UI 메서드 누락 및 클래스 속성 불일치 문제를 해결하고, 컨텍스트 메뉴 기능을 수복하여 전체 시뮬레이션 파이프라인의 완성도를 높였습니다.

## 2. 주요 변경 사항

### 2.1. 데이터 모델 및 분석 엔진 (`ShellDeformationAnalyzer`)
- **속성 표준화**: 기존의 `lx, ly` 인자명을 대시보드와 일관되게 `W, H` (Width, Height)로 변경했습니다.
- **데이터 형상(Shape) 유연성**: 입력 데이터(`m_raw`)의 형상을 JAX 배치 해석에 적합하게 자동 정규화하는 로직을 추가했습니다.
- **PCA 기반 자율 좌표계**: 주성분 분석(PCA)을 통해 판의 주평면을 찾고 로컬 좌표계를 생성하는 기능을 강화했습니다.

### 2.2. 시각화 레이어 (`QtVisualizerV2`)
- **UI 수복**: 누락되었던 `_init_3d_panel` 메서드를 재구현하여 VTK 입터랙터와 3D 제어 위젯들을 복구했습니다.
- **컨텍스트 메뉴 복구**: 3D 뷰어(`v_int`)에서 마우스 우클릭 시 파트별 가시성 제어 등을 위한 컨텍스트 메뉴가 표시되도록 정책 및 시그널을 연결했습니다.
- **속성 동기화**: `n_markers` 등 분석기 객체의 속성 참조 오류를 해결하여 렌더링 파이프라인의 안정성을 확보했습니다.

## 3. 검증 결과

### 3.1. 독립 실행 테스트(Standalone Demo)
- **명령**: `python plate_by_markers_v2.py`
- **결과**: 
    - 6개 파트에 대한 JAX 가속 해석 성공 (Avg RMSE: ~3.11e-11 mm)
    - 대시보드 GUI 및 컨텍스트 메뉴 기동 확인.
