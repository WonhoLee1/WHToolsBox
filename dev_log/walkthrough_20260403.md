# WHTOOLS Performance Analyst (v8-Scalable)

**WHTOOLS**입니다. 수천 프레임 이상의 대용량 데이터를 다루는 미래의 실험 환경을 대비하여, 시스템의 **확장성(Scalability)**을 극대화하는 작업을 완료했습니다.

## 1. 대용량 대응 기술 (Scalable Technologies)

### 1.1. Mini-batching 엔진 도입
- **기술적 배경**: JAX의 강력한 벡터화 연산은 빠르지만, 한 번에 수만 프레임을 처리할 경우 시스템 메모리(VRAM/RAM)가 일시적으로 부족해질 위험이 있습니다.
- **해결책**: 전체 데이터를 일정한 크기(기본 512프레임)의 배구러 나누어 순차적으로 처리하는 **Mini-batching** 로직을 `run_analysis`에 구현했습니다.
- **효과**: 메모리 점유율을 일정하게 유지하면서도 JAX의 병렬 연산 속도를 100% 활용할 수 있습니다.

### 1.2. 터미널 텍스트 진행 바 (Live Progress Bar)
- **사용자 경험**: 장시간 소요될 수 있는 대규모 해석 작업 시, 현재 진행 상태를 % 단위로 실시간 표시합니다.
- **구현 방식**: 외부 라이브러리 없이 순수 Python의 `sys.stdout` 제어를 통해 가벼우면서도 직관적인 진행 바를 구현했습니다.
- **로그 예시**: `Progress: |██████████████████████████████| 100.0% (1000/1000)`
