# [Walkthrough] Config UI 개편 및 재생 속도 동기화 고도화

안녕하세요, **WHTOOLS**입니다. 
시뮬레이션 초기 설정의 편의성 강화와 재생 속도 제어의 정확성을 확보하기 위해 요청하신 UI/UX 고도화 작업을 완료했습니다.

## 주요 변경 및 개선 사항

### 1. Config UI First (기동 순서 최적화)
- **우선 로드**: 시뮬레이션 실행 시 MuJoCo 뷰어가 뜨기 전, `ConfigEditor`가 가장 먼저 화면에 나타납니다.
- **사전 설정**: 사용자가 낙하 높이, 방향, 솔버 옵션 등을 완벽히 검토하고 `Apply`를 누르기 전까지 시뮬레이션이 대기하므로, 불필요한 시뮬레이션 재시작 횟수를 줄일 수 있습니다.

### 2. 탭(Tab) 기반 그룹화 UI 개편
- **ttk.Notebook 도입**: 방대한 설정 리스트를 7개의 기능별 탭으로 정교하게 분리했습니다.
    - **Scenario**: 낙하 높이, 모드, 방향 등 시나리오 조건
    - **Solver**: 수치 적분기, 타임스텝, 반복 횟수 등 물리 엔진 옵션
    - **Material**: 항복 강도, 소성 변형, 공기 저항 등 재질 특성
    - **Contact**: 각 부품 간의 접촉 강성 및 감쇠 파라미터
    - **Geometry**: 모델의 치수 및 격자 조밀도
    - **Mass**: 각 구성품의 질량 및 목표 중량 설정
    - **Others**: 내부 변수 및 기타 고급 설정
- **스크롤 최적화**: 각 탭 내부에서도 항목이 많을 경우 독립적인 스크롤을 지원하며, 마우스 휠 바인딩을 해당 영역 내부에서만 작동하도록 정밀하게 튜닝했습니다.

### 3. MuJoCo 방식의 실시간 재생 속도(Time Scale) 동기화
- **정확한 배속 제어**: 기존의 시뮬레이션 내부 스텝을 늘리던 방식에서, 실제 경과 시간(Wall-clock time)과 시뮬레이션 시간의 비율을 맞추는 **Real-time Sync** 로직으로 전환했습니다.
- **애니메이션 지연**: `time_scale`을 0.5x로 설정하면 물리 연산의 정밀도는 그대로 유지되면서 애니메이션만 2배 느리게 재생(Slow-motion)됩니다.
- **가속 모드**: 1.0x 이상의 배속에서는 CPU 성능 내에서 최대한 물리 연산을 가속 수행하여 시뮬레이션 완료 시간을 단축합니다.

## 검증 결과 요약

- [x] **기동 순서**: `enable_UI=True` 시 설정 창이 뷰어보다 먼저 뜨는 것 확인.
- [x] **탭 전환**: 7개 탭 간의 이동 및 각 탭 내 파라미터 렌더링 무결성 확인.
- [x] **속도 제어**: 0.1x (초고속 슬로우 모션) ~ 4.0x (가속) 구간에서 부드러운 재생 확인.

---

> [!tip] **사용 팁**
> 이제 마우스 휠을 사용하여 탭 내부의 설정을 빠르게 훑어볼 수 있습니다. 또한 시뮬레이션 도중 제어 창에서 `Time Scale`을 조정하면 MuJoCo의 `[` / `]` 키를 누른 것과 동일하게 즉각적으로 재생 속도가 변화합니다.

사용자 중심의 통합 시뮬레이션 환경 구축이 완료되었습니다. 추가적인 개선 사안이 있다면 언제든 말씀해 주세요!
