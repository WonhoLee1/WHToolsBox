# [v4.3.8] ConfigEditor 정교화 및 전수 피드백 반영 계획

안녕하세요, **WHTOOLS**입니다. 
UI의 완성도와 설명의 전문성에 대한 따끔한 지적 감사드립니다. 제가 기술적인 구현에만 매몰되어 사용자 경험과 '엔지니어링 도구'로서의 품격을 놓쳤습니다. 

이번 업데이트에서는 모든 설정 항목의 물리적 의미를 한글로 상세히 기술하고, Post-Processing UI와 완벽히 동일한 테마 및 폰트 환경을 테마 변경 시에도 유지되도록 구현하겠습니다.

## Proposed Changes

### 1. 전역 스타일 및 테마-폰트 동기화 (whts_gui.py)

- **테마 일치**: 기본 테마를 `breeze`로 설정하고, 모든 하부 위젯의 폰트를 `D2Coding` (1순위), `맑은 고딕` (2순위), **9pt**로 강제 지정합니다.
- **영구성 확보**: 테마 변경 기능 실행 시에도 위 폰트 설정이 즉시 재적용되도록 `_apply_custom_styles` 로직을 고도화하여 테마 스위칭 중에도 가독성을 유지합니다.

### 2. 불리언(Boolean) 입력 방식 개선

- **Combo 박스 도입**: `True/False` 값을 갖는 설정 항목(예: `enable_plasticity`, `include_cushion`)은 직접 입력을 지양하고 `ttk.Combobox`를 사용하여 선택하도록 변경합니다.

### 3. 고품질 한글 엔지니어링 가이드 (desc_map)

- **전수 한글화**: `whtb_config.py`의 주석 및 물리 엔진 이론을 바탕으로 모든 파라미터의 역할을 한글로 상세히 기술합니다. (단순히 "Parameter: ~~" 식의 성의 없는 표시를 제거합니다.)
    - 예: `cush_solref_stiff` -> "완충재의 수직 접촉 강성 (값이 작을수록 유연한 충격 흡수 모사)"
    - 예: `sim_impratio` -> "충격량 전달 비율 (접촉 시 에너지 전이 강도 제어)"

### 4. 제어(Control) 페이지 프리미엄 디자인

- **섹션별 분리 및 강조**:
    - **Header**: 실시간 상태를 가장 눈에 띄게 배치 (큰 폰트의 시간 정보 및 테마가 가미된 프로그레스 바).
    - **Body**: 재생 제어 및 속도 조절을 직관적인 아이콘 형태의 텍스트와 함께 배치.
    - **Footer**: 단축키 가이드를 세련된 스타일의 읽기 전용 텍스트 영역으로 구현하여 도움말 시인성 확보.
