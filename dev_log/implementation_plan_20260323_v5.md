# [구현 계획서] 리셋 기능 고도화 및 시각 효과 개선 (2026-03-23)

안녕하세요, **WHTOOLS**입니다.

시뮬레이션 사용자 경험(UX)을 개선하기 위해 리셋 로직 보강, 카메라 뷰 최적화, 그리고 지면 텍스트 변경 작업을 수행하겠습니다.

## 1. 개요
현재 시뮬레이션은 Backspace 키로 리셋 시 운동 상태만 초기화되고 영구 변형(Plasticity) 데이터가 유지되는 한계가 있습니다. 또한, 초기 카메라 뷰가 너무 가까워 모델 전체를 파악하기 어렵고, 지면의 흰색 배경이 단조로운 점을 개선하고자 합니다.

## 2. 세부 변경 사항
### 2.1. 리셋(Reset) 로직 강화 (`run_drop_simulation.py`)
- [ ] 시뮬레이션 시작 시 원본 지오메트리 정보(`geom_pos`, `geom_size`, `geom_rgba`)를 별도 저장.
- [ ] Backspace(R) 리셋 시 `mj_resetData`뿐만 아니라 저장된 원본 지오메트리 정보를 `model`에 다시 덮어씌워 영구 변형을 완전히 초기화.
- [ ] 영구 변형 상태 추적기(`geom_state_tracker`)를 클리어하여 새로운 낙하 테스트 준비.

### 2.2. 시각 효과 및 카메라 최적화 (`run_discrete_builder/__init__.py`)
- [ ] **ISO 카메라 추가**: `worldbody` 시작 부분에 모델 전체를 조망할 수 있는 `iso_view` 카메라 정의.
- [ ] **체크 무늬 지면 도입**: MuJoCo 표준인 `checker` 텍스처와 `ground_mat` 재질을 `asset`에 추가하고, 지면(`ground`)에 적용하여 공간감 및 시인성 확보.

## 3. 기대 결과
- 한 번의 리셋으로 시뮬레이션 상태뿐만 아니라 부품의 변형까지 초기화되어 반복적인 테스트가 훨씬 용이해집니다.
- 시뮬레이션 시작 즉시 전체 형상을 파악할 수 있는 최적의 뷰를 제공합니다.
- 지격자 무늬 지면을 통해 모델의 낙하 속도와 변위를 시각적으로 더 명확하게 인지할 수 있습니다.
