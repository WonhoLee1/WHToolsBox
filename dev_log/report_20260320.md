# MuJoCo Drop Simulation 업데이트 리포트 (2026.03.20)

어제 저녁부터 오늘까지 진행된 `WHToolsBox`의 관성 보정 고도화 및 영구 변형(Plastic Deformation) 시뮬레이션 기능 구현 사항을 정리한 보고서입니다.

---

## 1. 관성 보정 시스템 (Inertia Balancer) 고도화
- **선택적 질량 보정 지원**: `calculate_required_aux_masses` 함수가 `None` 입력을 허용하도록 개선되었습니다. Target Mass, CoG, MoI 중 특정 항목에 `None`을 입력하면 해당 항목은 보정하지 않고 Baseline 값을 유지한 채 나머지 요소만 최적화하여 보정용 Aux Mass를 산출합니다.
- **다중 보정 질량 배치**: 목표 관성치를 달성하기 위해 `chassis_aux_masses`에 다수의 블록을 자동 생성하고 배치하는 로직을 강화했습니다.
- **로깅 및 리포트 정밀화**: 시뮬레이션 시작 전 `Configuration Summary` 단계에서 보정 전(Baseline)과 보정 후의 관성 수치를 명확히 대조하여 출력하도록 리포트 형식을 개선했습니다.

## 2. 영구 변형 (Plastic Deformation) 시뮬레이션 구현
- **실시간 소성 변형 로직 도입**: 쿠션 요소가 강한 충격을 받아 바닥에 침투(Penetration)했을 때, 튕겨져 나오는 회복 단계에서 일정 비율(`plasticity_ratio`)만큼 영구적으로 찌그러지게 하는 물리 로직을 `run_drop_simulation.py`에 통합했습니다.
- **3D 변형 메커니즘 (Shift + Shrink)**: 단순히 위치만 옮기는 것이 아니라, 인접한 다른 패드와의 간섭을 방지하기 위해 **위치 이동(80%)**과 **크기 축소(20%)**를 동시에 적용하는 정밀 변형 방식을 채택했습니다.
- **공간적 제한 (Corner/Edge Constraint)**: 사용자의 피드백을 반영하여, 면 전체에 변형이 일어나는 비현실적인 상황을 방지하고 오직 **8곳의 모서리(Corner) 및 두께 방향 능(Edge Column)**에 위치한 패드들만 영구 변형이 발생하도록 범위를 제한했습니다.

## 3. 시각적 피드백 및 디버깅 강화
- **영구 변형 색상 연동 (Blue Tinting)**: 영구 변형이 발생한 블록은 즉시 **푸른색(Blue)**으로 색상이 변하도록 하여, 뷰어 상에서 어느 부위가 얼마나 찌그러졌는지 육안으로 즉시 식별할 수 있게 개선했습니다.
- **실시간 터미널 로그**: 변형이 발생할 때마다 해당 블록의 이름과 변형량(mm 단위)을 터미널에 실시간으로 출력하여 데이터 정합성을 검토할 수 있게 했습니다.

## 4. 시뮬레이션 안정성 및 버그 수정
- **접촉 감지 버그 해결**: Geom 네이밍 컨벤션(`g_bcushion` vs `b_bcushion`) 불일치로 인해 침투량이 감지되지 않던 문제를 수정하여 영구 변형 로직이 정상 작동하도록 했습니다.
- **조기 종료 시 충돌 방지**: 시뮬레이션 도중 ESC 등으로 신속히 종료할 때 인덱스 부족으로 `np.gradient`에서 발생하던 `ValueError`를 방지하는 방어 코드를 추가했습니다.
- **물리 파라미터 튜닝**: `Euler` 통합기(Integrator) 사용 및 타임스텝 조정을 통해 소성 변형이 수반되는 고속 충격 상황에서의 수치적 안정성을 확보했습니다.

**[결론]**
물질의 탄성 회복을 넘어선 실제적인 영구 변형 현상을 시뮬레이션에 성공적으로 이식하였으며, 특히 시각적/로그 데이터 피드백을 통해 해석 결과의 직관성을 획기적으로 높인 단계입니다.
