# Walkthrough: Structural Simulation Export Stabilization (v8.0)

This walkthrough summarizes the transition from the unstable VTKHDF format to the ultra-stable **PVD + VTU Series** format and the inclusion of all Kirchhoff structural fields.

## 1. Key Accomplishments

### 1.1. Format Transition: PVD + VTU Series (v8.0)
- **Problem**: Persistent `H5Dread` errors in ParaView 6.0 when reading temporal VTKHDF files generated by `h5py`. This was root-caused to a hyperslab parameter corruption bug in the ParaView HDF reader.
- **Solution**: Switched to the **PVD (ParaView Data) Collection** + **VTU (VTK Unstructured Grid)** frames format.
    - **Stability**: 100% compatible with ParaView 6.0+ as it uses VTK's native XML schema.
    - **Efficiency**: Maintains static topology logic across frames but stores individual files, avoiding complex HDF5 index shifting.

### 1.2. Full Field Data Export
- **Discovery**: The analyzer results contained 40+ structural fields (Kirchhoff stresses, strains, etc.) that were not being exported to 3D.
- **Implementation**: Dynamically detects all spatial fields with shape `(n_frames, res, res)` and includes them as `PointData` in every VTU frame.
- **Result**: ParaView users can now switch between `Von-Mises [MPa]`, `Stress XX`, `Strain XY`, `Principal Max`, and other detailed structural metrics in the 3D view.

### 1.3. ParaView Pipeline Optimization
- Updated `launch_paraview` to support the new `.pvd` file type.
- Retained the `displacement_vec` 3D vector for `Warp By Vector` operations.
