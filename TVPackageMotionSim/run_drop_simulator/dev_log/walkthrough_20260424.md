# Walkthrough - LTL Orientation Mapping, UI Control, Recording Fix & Real-Time Monitor Enhancements (2026-04-24)

## 1. 개요
본 작업은 LTL 설정 업데이트, 시뮬레이션 제어 로직 개선 및 실시간 모니터링(`Real-Time Monitor`) 기능의 대대적인 업그레이드를 포함합니다. 특히 타임라인 슬라이더와의 연동성 및 UI 안정성을 강화했습니다.

## 2. 변경 사항 세부 정보

### 2.1 whtb_utils.py 수정 (LTL 맵핑)
`parse_drop_target` 내의 `ltl_map`을 수정하여 면 번호에 따른 벡터 방향을 재정의했습니다. (1:Top, 2:Back, 3:Bottom, 4:Front)

### 2.2 whts_engine.py 수정 (고급 물리 데이터 수집)
- **강체 거동 대표 물리량**: 회전축, 회전 속도, 병진 속도를 매 스텝 저장합니다.
- **슬라이더 내비게이션**: 멈춤 상태에서 슬라이더를 움직이면 해당 프레임의 스냅샷으로 즉시 점프하며 뷰어를 갱신합니다.

### 2.3 whts_monitor.py 수정 (오류 수정 및 슬라이더 연동)
- **PySide6 호환성 수정**: 특정 환경에서 `Qt.WindowIconHint`를 찾지 못하는 `AttributeError`를 해결하기 위해 윈도우 플래그 설정을 보다 안정적인 방식으로 변경했습니다.
- **실시간 타임 마커 (Time Marker)**:
    - 모니터링 그래프에 현재 시뮬레이션 시간을 나타내는 **빨간색 점선(Vertical Line)**을 추가했습니다.
    - 슬라이더를 움직여 특정 과거 시점으로 점프하면, 그래프 상의 마커도 해당 시점으로 함께 이동하여 **데이터와 물리 현상 간의 동기화**를 직관적으로 확인할 수 있습니다.
- **지표 확장**: 병진 속도 다축 시각화, 방위각/고도각 분석 기능을 포함합니다.

## 3. 검증 (Verification)
- `Monitor` 버튼 클릭 시 더 이상 에러가 발생하지 않고 정상적으로 설정창이 뜹니다.
- 시뮬레이션을 정지한 후 타임라인 슬라이더를 앞뒤로 움직였을 때, 모니터 그래프의 빨간색 세로선이 실시간으로 따라 움직이며 뷰어의 형상과 데이터 시점이 일치함을 확인했습니다.
