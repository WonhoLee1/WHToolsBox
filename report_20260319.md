# MuJoCo Drop Simulation 업데이트 리포트 (2026.03.19)

오늘 진행된 시뮬레이션 환경(WHToolsBox) 개선 및 추가 기능 구현 사항들을 종합적으로 정리한 보고서입니다.

---

## 1. 데이터 출력 및 내보내기 고도화 (Excel/TXT 통합)
- **TXT 파일 데이터 포맷 개선**: `rds-[시간]_data.txt` 출력 시 범례(행/열 제목)가 20칸을 넘어가는 경우 자동으로 줄바꿈(wrap)하여 주석 마킹(`#`)과 함께 안전하게 누적되도록 개선했습니다.
- **Excel 파일 자동 생성 (`openpyxl`)**: 시뮬레이션 그래프(PNG)의 데이터를 시트(Sheet) 별로 분리하여 저장하는 기능을 도입했습니다. `rds-[시간]_data.xlsx` 파일 내에서 상단 공간(A1)에 추출된 결과 그래프(PNG) 이미지를 바로 복사해 오고, 그 아래쪽(10행 이하)부터 txt와 완벽히 유사한 형태의 데이터 시계열 테이블을 펼쳐놓도록 구성했습니다.
- **True_COG 데이터 트래킹 및 보강**: 무의미했던 기하학적 기준점(Center)과 별개로 실제 조립체 전체의 진무게중심 좌표(True_COG)를 추적하여 추출 리스트 및 플로팅 시스템에 추가했습니다.

## 2. 향후 최적화 연동을 위한 Python 데이터 객체화 (`DropSimResult`)
- 향후 "실제 실험 좌표와 시뮬레이션 좌표의 오차 최적화 매칭(Parameter Optimization)" 시스템 연동을 미리 염두에 두고, 모든 데이터를 담은 **`DropSimResult` 캡슐화 데이터 클래스**를 설계 및 도입했습니다.
- 시뮬레이션 구동 종료 시, 즉시 `rds-[시간]_result.pkl` 파일로 직렬화(Serialize)되어 저장됩니다. 
- 이후 최적화 루틴에서 단 한 줄의 코드(`DropSimResult.load(...)`)만으로 8개 위치의 Corner 트래킹 정보들(`corner_pos_hist`)에 언제든 자유롭게 접근하여 에러 매칭(Loss calculation)이 가능합니다.

## 3. 물리 엔진 충돌 및 렌더링(Viewer) 시스템 최적화
- **렌더링 그룹핑(Geometry Groups) 적용**: 
  조립체의 복잡도가 증가함에 따라 육안(GUI Viewer) 디버깅 및 관측 효율을 획기적으로 상승시키기 위해 지오메트리 `<geom>` 요소들에 `group` 속성을 부여했습니다.
  - **Group 0**: Ground(바닥) 및 여타 기본 기구물
  - **Group 1**: Cushion (완충재 전용)
  - **Group 2**: TV 및 섀시 계통 (chassis, opencell, adhesive, tape 등)
  - **Group 3**: Aux Masses (스피커 등 임의 추가 질량)
  - *효과*: 이제 뷰어 실행 시 키보드 숫자키 `0 ~ 3`을 통해 원하는 계통의 부품만 즉각적으로 숨기거나 렌더링할 수 있습니다.
- **추가 질량(Aux Mass) 조립 충돌(Weld) 버그 해결**: 
  오프셋을 준 스피커 등의 추가 질량체가 용접부(weld)의 강제 스내핑(`site1~site2`) 현상으로 인해 블랙홀처럼 쿠션 중앙으로 거칠게 잡아당겨지며 쿠션을 찌그러뜨리던 현상을 완벽히 해결했습니다. 조인 방식을 초기 오프셋을 절대적으로 유지하는 `body1~body2` 용접 구속 조건으로 전환하였으며, 자체 `contype="0"` 옵션이 철저히 지켜져 어떠한 구조적 파괴 없이 순수 관성에만 완벽히 개입되도록 수정했습니다.

## 4. UI/그래픽 레이아웃 수정 및 Viewer 오류 방지
- **그래프 겹침 버그 개선**: 블록별 상세 변형(Deformation All) 그래프에서 범례(Legend)와 3D 인덱스 참조도(i-j-k Reference Layout)가 서로 겹쳐서 난잡하게 렌더링 되던 UI 위치를 각 모서리 끝단으로 명확히 밀어내어 직관성을 높였습니다.
- **시뮬레이션 초기화(리셋) 버그 픽스**: Viewer 상호작용 도중 백스페이스(`Backspace`)를 누르면 이전 저장 리스트들이 꼬이면서 충돌(`AttributeError`)하던 이슈를 방지하고자 `.clear()` 메서드가 dict와 array 계층 모두 안전하게 도달하도록 오류를 잡았습니다.
